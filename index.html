INSTALL

Bun · OpenCode 자동 설치 · ~/.local/bin 등록 · 소스 보기 ↗

WHY
분산 실행

Mac에서 orchestrator, GPU 서버에서 builder, NAS에서 verifier를 동시에 실행. 머신 경계를 투명하게 넘는다.

HOW
Phoenix Channel

Elixir Phoenix WebSocket을 L1 메시지 버스로 사용. task.assign → task.result 이벤트가 채널을 통해 브로드캐스트.

WHAT
Work Key

세션마다 LN-YYYYMMDD-XXX Work Key가 발급된다. 모든 에이전트가 같은 채널에 JOIN해 협업한다.

03

채널 이벤트

EVENT DIRECTION PAYLOAD (주요 필드) 설명
task.assign orch → builder to, task_id, instructions 태스크 할당. builder가 OpenCode 실행.
task.progress builder → all task_id, message, output_tail 10초마다 진행 상황 브로드캐스트.
task.result builder → all task_id, exit_code, artifacts[] 태스크 완료. ARTIFACT: 라인 파싱됨.
task.blocked any → all task_id, reason 진행 불가. orchestrator가 재배정 판단.
task.approval_requested any → orch task_id, action, rationale 인간 승인 게이트 (ACP).
state.update any → server key: value pairs ETS Work Key 상태 업데이트.
mailbox.post any → target to, body 오프라인이면 큐잉, 온라인이면 즉시 전달.
agent.hello server → all agent, role, machine 에이전트 JOIN 알림.
agent.bye server → all agent, role 에이전트 퇴장 알림.

REST API

METHOD ENDPOINT 응답
GET /api/health {"ok":true,"work_keys":N}
GET /api/presence 온라인 에이전트 목록
GET /api/work-keys/latest 최신 Work Key (없으면 404)
POST /api/work-keys 새 Work Key 발급
GET /api/state/:work_key Work Key 상태 조회
PATCH /api/state/:work_key Work Key 상태 업데이트
GET /api/mailbox/:agent 오프라인 메시지 수신 (큐 삭제)
