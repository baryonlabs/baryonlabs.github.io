open-agent-harness

여러 머신의 AI 에이전트가 NAS 공유 workspace에서 협업해
하나의 소프트웨어를 병렬로 개발한다.
에이전트 추가는 명령어 한 줄이면 충분하다.

INSTALL
curl -fsSL https://baryonlabs.github.io/install.sh | bash

Bun · OpenCode 자동 설치 · ~/.local/bin 등록 · 소스 보기 ↗

STORAGE
NAS 공유 Workspace

모든 에이전트가 동일한 NAS 경로를 마운트. 파일을 메시지로 복붙할 필요 없이 직접 읽고 쓴다.

NETWORK
Phoenix Channel

Elixir Phoenix WebSocket이 조율 레이어. task.assign → task.result 이벤트가 채널을 통해 실시간 브로드캐스트.

PARALLEL
병렬 개발

builder들이 서로 다른 모듈을 동시에 개발. integrator가 NAS에서 직접 읽어 통합. 단일 에이전트 대비 시간 단축.

SCALE
원라인 확장

새 머신에서 명령어 한 줄로 에이전트 추가. Bun·OpenCode 자동 설치, Work Key 자동 감지, 즉시 협업 참여.

05

에이전트 추가

새 머신을 협업에 참여시키려면 명령어 한 줄이면 된다. Bun · OpenCode가 없으면 자동 설치하고, Work Key는 서버에서 자동으로 감지한다.

builder
코드 작성
bash scripts/setup-agent.sh ws://서버IP:4000 builder "" /workspace
verifier
테스트 실행
bash scripts/setup-agent.sh ws://서버IP:4000 verifier "" /workspace
reviewer
코드 리뷰
bash scripts/setup-agent.sh ws://서버IP:4000 reviewer "" /workspace
신규 설치
자동 설치 포함
ROLE=builder DIR=/workspace bash <(curl -fsSL https://raw.githubusercontent.com/baryonlabs/open-agent-harness/main/scripts/setup-agent.sh) ws://서버IP:4000

Work Key(LN-YYYYMMDD-XXX) 생략 시 서버에서 최신 WK 자동 감지 · AGENT_NAME 기본값: role@hostname

06

채널 이벤트

EVENTDIRECTIONPAYLOAD (주요 필드)설명
task.assignorch → agentto, task_id, role, instructions태스크 할당. 대상 agent가 OpenCode 실행.
task.progressagent → alltask_id, message, output_tail10초마다 진행 상황 브로드캐스트.
task.resultagent → alltask_id, exit_code, artifacts[]태스크 완료. ARTIFACT: 라인 자동 파싱.
task.blockedany → alltask_id, reason진행 불가. orchestrator가 재배정 판단.
state.updateany → serverkey: valueWork Key 상태 업데이트 (ETS).
mailbox.postany → targetto, body오프라인이면 큐잉, 온라인이면 즉시 전달.
agent.helloserver → allagent, role, machine에이전트 JOIN 알림.
agent.byeserver → allagent, role에이전트 퇴장 알림.
